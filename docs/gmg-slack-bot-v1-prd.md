# GMG Slack Bot v1 PRD

## 제품 목표

GMG는 사람들이 Slack에서 흩어진 사람들에게 일일이 물어보지 않고도 번개 모임을 만들고 참여자를 모을 수 있게 해준다. 첫 버전은 모임 생성, 참여 응답, 마감 기반 확정을 쉽고 잘 보이게 만드는 데 집중한다.

## 문제

현재는 모임 참여자를 구할 때 대화로 직접 물어봐야 한다. 사람들이 여러 채널에 흩어져 있고 Slack 알림도 분산되어 있어서 빠르게 모임 인원을 모으기 어렵다. GMG는 이 모집과 상태 확인 흐름을 Slack 안에서 자동화한다.

## 대상 사용자

- Creator(생성자): `/gmg`로 모임을 시작하는 사용자.
- Participant(참여자): 모임에 응답하는 사용자.
- Considering user(고민 중인 사용자): 아직 결정하지 않고 고민 중인 사용자.
- Non-attendee(불참자): 불참 의사를 명확히 표시한 사용자.

## 포함 범위

- `/gmg` slash command로 모임 생성.
- 모임 정보를 입력하는 Slack modal.
- 모임 정보와 참여 버튼이 포함된 announcement-channel message.
- 참여 상태:
  - `GMG`
  - `late_join`
  - `considering`
  - `not_attending`
  - 응답 취소
- 선택적 정원:
  - 정원 있는 모임.
  - 정원 없는 모임.
- 자동 확정:
  - 정원 도달.
  - 마감 시간 도달.
- Creator의 수동 확정과 취소.
- 리마인더 메시지:
  - 모임 확정.
  - 모임 취소.
  - 약속 시간 임박.
  - 약속 30분 전, 아직 `considering` 상태인 사용자 언급.

## 제외 범위

- 비용 정산.
- 캘린더 연동.
- 승인, 검열, 운영자 심사.
- 참석률 통계.
- 식당 또는 장소 추천.
- 결제, 보증금, 벌금.
- 여러 Slack workspace 배포.
- 별도 웹 앱 또는 모바일 앱.

## 사용자 이야기

### 모임 만들기

Creator는 어느 채널에서든 `/gmg`를 실행하고 모달을 작성한 뒤, 공유 announcement channel에 모임을 게시할 수 있다.

완료 기준:

- command는 slash command를 지원하는 일반 Slack 대화에서 동작한다.
- modal은 제목, 종류, 약속 시간, 선택적 정원, 마감 시간을 입력받는다.
- 공개 공지는 source channel이 아니라 설정된 announcement channel에 올라간다.

### 참여 또는 응답하기

Participant는 마감 전까지 `GMG`, `late_join`, `considering`, `not_attending` 중 하나를 선택하거나 응답을 취소할 수 있다.

완료 기준:

- 한 사용자는 한 모임에서 하나의 active status만 가진다.
- 새 status를 선택하면 이전 status가 덮어써진다.
- `GMG`와 `late_join`은 정원에 포함된다.
- `considering`과 `not_attending`은 정원에 포함되지 않는다.

### 확정 또는 취소하기

Creator는 모임을 수동으로 확정하거나 취소할 수 있다.

완료 기준:

- 수동 확정은 모임을 잠그고 이후 참여자 변경을 막는다.
- 취소는 모임을 잠그고, 공지를 갱신하며, 취소 리마인더를 보낸다.

### 자동 최종 확정

Participant는 마감 시간이 되면 모임이 최종 상태로 확정된다고 믿을 수 있어야 한다.

완료 기준:

- 정원이 없는 모임은 마감 전까지 계속 참여할 수 있다.
- 마감 시간이 되면 현재 `GMG + late_join` 사용자가 최종 참석자가 된다.
- 마감 후에는 참여 상태 변경을 거절한다.

## 제품 톤

GMG는 특히 `considering` 사용자를 향한 30분 전 리마인더에서 장난스러운 톤을 사용할 수 있다. 단, 전체 톤은 친근하고 상태 중심이어야 한다. insult, shame, 개인 특성 언급은 하지 않는다.
